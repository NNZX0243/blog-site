    FRP内网穿透

    发表于 2025-06-28 更新于 2025-06-28 分类于 Other

    1 准备工作

    • 首先你得有台云服务器部署frp服务，这里就不讲怎么去开云服务器了。站长的云服务器系统使用的是ubuntu
    • 去GitHub下载frp工具
    • 我目前使用的版本是v0.62.1
    • 下载frp_0.62.1_linux_amd64.tar.gz（云服务器上作为服务端用）和frp_0.62.1_windows_amd64.zip（windows客户端用）
    • 注意：下载frp工具杀毒软件和浏览器都会报毒，忽略就行

    2 官方文档

    • frp

    3 云端部署frp服务

    • 下面操作提示没权限的话自己 sudo提权

    3.1 建立文件夹

    bash 
    mkdir -p /home/frp
cd /home/frp

    3.2 下载frp

    bash 
    wget https://github.com/fatedier/frp/releases/download/v0.62.1/frp_0.62.1_linux_amd64.tar.gz
tar -zxvf frp_0.62.1_linux_amd64.tar.gz
cd frp_0.62.1_linux_amd64
ls #一般会有下面4个文件 
frpc #客户端程序，这里不必管它
frpc.toml #客户端程序配置文件，这里不必管它
frps #服务端程序，待会启动它
frps.toml #服务端程序配置文件
LICENSE # 不用管

    3.3 编辑frp服务端配置文件

    bash 
    vim frps.toml
    • 填入下面内容wq退出,
    • 注意：把#号注释删除
    txt 
    bindAddr = "0.0.0.0"# 监听地址，这里表示监听本服务器IP地址
bindPort = 7100#frp服务端口

# 下面内容是启动一个webServer，通过浏览器输入 服务器IP：7500可以图形化查看frp状态
webServer.addr = "0.0.0.0"# 监听地址，这里表示监听本服务器IP地址
webServer.port = 7500# webServer服务端口
webServer.user = "username"# 用户名，用于登录
webServer.password = "password"# 密码，用于登录

# 下面设置frp log
log.to = "./frps.log"# log文件地址
log.level = "info"
log.maxDays = 5

auth.method = "token"# 启用token验证 
auth.token = "21345679"# token， 客户端要配置和这里一样的token才能正常内网穿透

# frp服务可以使用的端口范围6000~7000
allowPorts = [
    { start = 6000, end = 7000}
]

    3.5 systemd配置

    • 在 Linux 系统下，使用 systemd 可以方便地控制 frps 服务端的启动、停止、配置后台运行以及开机自启动。

    以下是具体的操作步骤：

    3.5.1 安装 systemd

    如果您的 Linux 服务器上尚未安装 systemd，可以使用包管理器如 yum（适用于 CentOS/RHEL）或 apt（适用于 Debian/Ubuntu）来安装它：

    bash 
    # 使用 yum 安装 systemd（CentOS/RHEL）
yum install systemd

# 使用 apt 安装 systemd（Debian/Ubuntu）
apt install systemd

    3.5.2 创建 frps.service 文件

    使用文本编辑器 (如 vim) 在 /etc/systemd/system 目录下创建一个 frps.service 文件，用于配置 frps 服务。

    bash 
    sudo vim /etc/systemd/system/frps.service

    写入内容

    bash 
    [Unit]
# 服务名称，可自定义
Description = frp server
After = network.target syslog.target
Wants = network.target

[Service]
Type = simple
# 启动frps的命令，需修改为您的frps的安装路径
ExecStart = /path/to/frps -c /path/to/frps.toml

[Install]
WantedBy = multi-user.target

    3.5.3 使用 systemd 命令管理 frps 服务

    bash 
    # 启动frp
sudo systemctl start frps
# 停止frp
sudo systemctl stop frps
# 重启frp
sudo systemctl restart frps
# 查看frp状态
sudo systemctl status frps

    3.5.4 设置 frps 开机自启动

    bash 
    sudo systemctl enable frps

    3.6 启动frps服务

    bash 
    ./frps -c ./frps.toml &# & 表示后台运行

    4、 云端frps脱离终端运行

    • 实际运行发现在终端上直接执行./frps -c ./frps.toml，终端关闭也会导致frps进程退出

    4.1 安装screen

    bash 
    sudo apt-get install screen

    4.2 创建screen会话

    bash 
    screen -S frpsscreen

    4.3 在screen会话中运行frps服务

    bash 
    ./frps -c ./frps.toml &

    4.4 剥离screen会话

    • Ctrl+A+D (同时按键盘上的 Ctrl、A、D 三个键)

    • 说明：此时可以关闭终端，让程序在服务器上继续运行了

    4.5 重新接入screen会话

    • 查看当前存在的screen会话
    bash 
    screen -ls

    4.6 接入会话

    bash 
    screen -x 进程号
    • 或者
    bash 
    screen -r frpsscreen

    4.7 关闭所有screen会话

    • 查看有多少个screen
    bash 
    screen -ls |awk '/Socket/'|awk '{print $1}'
    • 如果有20个screen，关闭所有screen的命令为
    bash 
    screen -ls|awk 'NR>=2&&NR<=20{print $1}'|awk '{print "screen -S "$1" -X quit"}'|sh

    5、本地客户端内网穿透

    5.1 建立文件夹

    • 随便找个文件夹放frp_0.62.1_windows_amd64.zip，解压
    • 一般会解压出5个文件
    bash 
    frpc #客户端程序
frpc.toml #客户端程序配置文件
frps #服务端程序，这里不必管它
frps.toml #服务端程序配置文件，这里不必管它
LICENSE #不用管

    5.2 编辑frp客户端配置文件

    • 使用任意编辑器打开frpc.toml编辑
    • 注意：把#号注释删除
    txt 
    serverAddr = "192.168.1.1"# frp服务器公网IP
serverPort = 7100# frp服务器监听的端口
loginFailExit = true

# 日志文件配置
log.to = "./frpc.log"
log.level = "info"
log.maxDays = 5

auth.method = "token"# token验证
auth.token = "21345679"# token， 客户端这里要和服务端一致

[[proxies]]
name = "NNZX BLOG"# 代理名称，随便取
type = "tcp"# 协议类型
localIP = "127.0.0.1"# 本机IP地址
localPort = 5500# 本机需要内网穿透的端口
remotePort = 6100# frp服务器暴露在公网上的端口，通过访问154.64.231.134：6100 可以访问到本机5500端口

[[proxies]]
name = "Minecraft"
type = "tcp"
localIP = "127.0.0.1"
localPort = 25565
remotePort = 6255

    5.3 启动内网穿透

    • 使用终端运行frpc.exe
    bash 
    ./frpc -c ./frpc.toml
